EBS can’t compile invalid objects

Sep 8, 2013

Recently, I’ve working on an patch to add ASCP module to my EBS. But when I was appling the patch and compiling the invalid objects, it just stucks.

At the first time, I thought it was just as normal and need some time,but after one-nights wait , I use the following SQL and just find the invalid objects just don’t get less.

select status,count(*) from dba_objects group by status;

Restart DB/Rebooting Server just didn’t help. So I checked the Database alert log file. Location:

$ORACLEHOME/admin/DEVshmdev/diag/rdbms/dev/SID/alert

And found the message which showed that the SYSAUX tablespace is not enough.

<msg time=’2013-01-18T14:06:02.569+08:00′ org_id=’oracle’ comp_id=’rdbms’ client_id=” type=’UNKNOWN’ level=’16′ host_id=’shmdev’ host_addr=’192.168.147.104′ module=” pid=’11937′>
<txt>
ORA-1655: unable to extend table SYS.SMON_SCN_TO_TIME_AUX by 128 in tablespace SYSAUX
</txt>
</msg>

It is easy to figure out that we should extend our tablespace. Use following SQL to check our tablespace usage:

select total.tablespace_name,
round(total.MB, 2) as Total_MB,
round(total.MB – free.MB, 2) as Used_MB,
round((1 – free.MB / total.MB) * 100, 2) || ‘%’ as Used_Pct
from 
(
    select tablespace_name, sum(bytes) / 1024 / 1024 as MB
    from dba_free_space
    group by tablespace_name
    ) free,
(
    select tablespace_name, sum(bytes) / 1024 / 1024 as MB
    from dba_data_files
    group by tablespace_name
    ) total
where free.tablespace_name = total.tablespace_name;

Extend the SYSAUX tablespace:

ALTER DATABASE DATAFILE ‘ /oracle/DEV/db/apps_st/data/sysaux01.dbf” AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED;

And now we can compile the objects easily.
