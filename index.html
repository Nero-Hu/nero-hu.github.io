Deploy Rime[Squirrel] on OS X

什么是中州韵

中州韵输入法引擎，思想用键盘表达也行。

如何安装

下载Squirrel(Rime Mac Version)并且安装.pkg文件。

基础设置

自定义输入配置文件

cd ~/Library/Rime
touch squirrel.custom.yaml

编辑squirrel.custom.yaml.

patch:
  style:
    horizontal: true
    font_point: 14

此配置将会覆盖默认安装目录下的default.yaml文件内的相应设置。

比如这里就将候选字体的大小设置为14，并且将候选模式设置为水平。

用户数据同步

cd ~/Library/Rime
vi installation.yaml

sync_dir: '/Dropbox/Rime-Settings'

** 修改上述设置后，需要重新部署squirrel才会生效。

参考:

  1. Rime home page
  2. Rime User Guide
  3. Rime Customization Guide

Trace the smallest element in a specially designed stack

Question:

Design a stack, which owns a min() function that returns the smallest elements within the stack.

Additional requirements

Time complexity of push(), pop() and min() function should be O(1).

Solution

Code

The point should be in the function of push() shown as following code.

public NodeWithMin push(NodeWithMin item) {
    // TODO Auto-generated method stub
    if (this.isEmpty()){
        item.MinNode = item.Node;
        return super.push(item);
    }
    else{
        if ( item.Node > this.peek().MinNode ) {
            item.MinNode = this.peek().MinNode;
            return super.push(item);
        }
        else{
            item.MinNode = item.Node;
            return super.push(item);
        }
    }
}

Thinking, Fast and Slow

Recently reading [Thinking, Fast and Slow] by Daniel Kahneman, and I have to admit that this book really changes my views again, as intensively as another book I’ve read [Logical Thinking] who describes a tremendous way to break down problems called MECE (Mutually Exclusive, Collectively Exhaustive).

I’ve not completed this book yet, but there are many useful infos when you think.

Two mechanisms

  • System 1: Fast, automatic, frequent, emotional, stereotypic, subconscious
  • System 2: Slow, effortful, infrequent, logical, calculating, conscious

We are borned lazy

There is always a tendency to keep lazy, not only in our mind, but also in everything in the world. Everything stays at or tends to stay at a lowest center of gravity in normal conditions, animals keep evolving to save more energy, etc… Human being following the nature rules, and are inclined not to make efforts, if the System 1 could help us save a big part of daily problems.

We are blind to obvious, and of course blind to our blindness

Our efforts are limited, which means you will be blind to the things that beyond your efforts boundary. It is “obvious” that you are always missing something.

Substitution

We are always substitute the problems we asked with a more comfortable question to us. E.g: We could simply judge a person’s characteristics through his appearance, because we use the problem “how do you like this guy” instead of the question we asked.

We can be easily manipulated

Even the most small muscle change will cause your change of view.

Do not think that you are immune to be shaked! It can be done even without your attention.

Overconfidence

People are overconfident, especially to the area they familiar with. Actually, we act worse in the area we know most, because we tend to believe ourselves, not the facts.

Ignore our ignorance

The world is hard to be predicted, what you see is not the way the world works.

Regression toward the mean

It is an amazing way to adjust your expectation. You can not expect a continuous extreme good/bad luck in your performance. Please refer to wiki for details.

EBS can’t compile invalid objects

Recently, I’ve working on an patch to add ASCP module to my EBS. But when I was appling the patch and compiling the invalid objects, it just stucks.

At the first time, I thought it was just as normal and need some time,but after one-nights wait , I use the following SQL and just find the invalid objects just don’t get less.

select status,count(*) from dba_objects group by status;

Restart DB/Rebooting Server just didn’t help. So I checked the Database alert log file. Location:

$ORACLEHOME/admin/DEVshmdev/diag/rdbms/dev/SID/alert

And found the message which showed that the SYSAUX tablespace is not enough.

<msg time=’2013-01-18T14:06:02.569+08:00′ org_id=’oracle’ comp_id=’rdbms’ client_id=” type=’UNKNOWN’ level=’16′ host_id=’shmdev’ host_addr=’192.168.147.104′ module=” pid=’11937′>
    <txt>
    ORA-1655: unable to extend table SYS.SMON_SCN_TO_TIME_AUX by 128 in tablespace SYSAUX
    </txt>
    </msg>

It is easy to figure out that we should extend our tablespace. Use following SQL to check our tablespace usage:

select total.tablespace_name,
    round(total.MB, 2) as Total_MB,
    round(total.MB – free.MB, 2) as Used_MB,
    round((1 – free.MB / total.MB) * 100, 2) || ‘%’ as Used_Pct
    from 
    (
        select tablespace_name, sum(bytes) / 1024 / 1024 as MB
        from dba_free_space
        group by tablespace_name
        ) free,
    (
        select tablespace_name, sum(bytes) / 1024 / 1024 as MB
        from dba_data_files
        group by tablespace_name
        ) total
    where free.tablespace_name = total.tablespace_name;

Extend the SYSAUX tablespace:

ALTER DATABASE DATAFILE '/oracle/DEV/db/apps_st/data/sysaux01.dbf' AUTOEXTEND ON NEXT 100M MAXSIZE UNLIMITED;

And now we can compile the objects easily.
